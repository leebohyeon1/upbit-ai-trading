# 클라우드 로깅 기능 비활성화 안내

## 중요 공지: 파이어베이스 기능이 제거되었습니다

비트코인 자동매매 프로그램에서 Firebase Cloud Firestore를 사용한 클라우드 로깅 기능이 제거되었습니다. 이제 프로그램은 로컬 파일 시스템에만 로그를 저장합니다.

## 기능 제거 내용

다음 파이어베이스 관련 기능들이 제거되었습니다:

1. Firebase Cloud Firestore 연결 및 로그 저장 기능
2. 클라우드 로깅 설정 및 관리 유틸리티
3. Firebase 관련 의존성 및 패키지

## 로그 저장 방법

이제 모든 로그는 다음 로컬 파일에만 저장됩니다:

1. `logs/app.log`: 애플리케이션 일반 로그
2. `logs/trade.log`: 거래 관련 로그
3. `logs/error.log`: 오류 로그

## 로그 확인 방법

로그를 확인하려면 `logs` 디렉토리에 있는 해당 로그 파일을 직접 확인하세요. 로그 파일은 일정 크기(10MB)에 도달하면 자동으로 백업되고 새 파일이 생성됩니다(최대 5개 백업 파일 유지).

```
logs/
├── app.log       # 최신 애플리케이션 로그
├── app.log.1     # 이전 애플리케이션 로그 백업
├── trade.log     # 최신 거래 로그
├── trade.log.1   # 이전 거래 로그 백업
├── error.log     # 최신 오류 로그
└── error.log.1   # 이전 오류 로그 백업
```

## 대체 솔루션 (필요한 경우)

클라우드 로깅이 필요한 경우 다음과 같은 대안을 고려할 수 있습니다:

1. 로그 파일을 클라우드 스토리지(Dropbox, Google Drive 등)에 동기화
2. 외부 로깅 서비스 통합(예: Datadog, Loggly, ELK 스택 등)

이러한 대체 솔루션을 적용하려면 별도의 개발이 필요합니다.
